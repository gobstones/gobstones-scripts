(static) create(projectName, projectTypeopt, packageManageropt)

Create a new library project in a subfolder with the given name. This includes creating all the required and recommended style files define a package.json, git configuration files, visual studio code files, NPM configuration files, contribution guidelines, a readme, a license (MIT by default), install all dependencies, and initialize a git repository.

Parameters:
Name Type Attributes Default Description
projectName string

The name of the project to be created.
projectType string <optional>
 library

The project type to create (Defaults to "library").
packageManager string <optional>

The package manager to use when downloading dependencies.
Throws:

  • If the argument for projectType or packageManager is invalid.

  • If there's already a folder with the project name, or the folder is not empty.

(static) eject(forceopt, file, projectTypeopt) → {Array.<string>}

Eject all the general configuration files to the root project. This includes configuration files for Typescript, Typedoc, JEST, Rollup, and nps. This command is intended to be run locally. If force is added, all previously created local files are updated to their latest version. If not, only missing files are copied.

Parameters:
Name Type Attributes Default Description
force boolean <optional>
 false

Wether to force the ejection of files, that is, override already present files in the project with their newest version (Defaults to false).
file string all

The file name to update, or "all" if all should be updated (Defaults to "all").
projectType string <optional>
 library

The project type to eject from (Defaults to "library").
Returns:

The list of updated files.

Type
Array.<string>

(static) init(projectTypeopt, packageManageropt)

The same as create, but runs on the local folder as a project. That is, initialize a project in the given folder, which default to the root. Note that the folder must be empty in order to initialize a project in the folder.

Parameters:
Name Type Attributes Default Description

 library

packageManager string <optional>

The package manager to use when downloading dependencies.
Throws:

If the current folder is not empty.

(static) update(forceopt, file, projectTypeopt) → {Array.<string>}

Override the missing configuration files that are created on an init or create command on the local project. This is intended to be run locally, on an already created project, to update the configuration. By appending force as a subcommand, all files are updated to their latest version.
